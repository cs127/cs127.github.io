IT/MPTM effect commands

cs127
last update: 2024-02-05

byte command subcommand behaviour notes memory
$00 xx — — —
$01 A Axx set ticks per row to $xx. behaviour of parameter $00 is undefined.
standard behaviour is to do nothing. 		—
$02 B Bxx jump to order $xx. jumps to row 0 by default, but can be used alongside Cxx to specify a different row.
OpenMPT: can be followed by #xx for extra digits. 		—
$03 C Cxx jump to row $xx. jumps to the next order by default, but can be used alongside Bxx to specify a different order.
OpenMPT: can be followed by #xx for extra digits. 		—
$04 D D0x slide note volume down by $x (coarse). if neither of the digits are $0 or $F, the behaviour is undefined.
standard behaviour is to do nothing. 		D (D, K, L)
Dx0 slide note volume up by $x (coarse).
DFx, $0 < $x < $F slide note volume down by $x (fine).
DxF, $0 < $x slide note volume up by $x (fine).
$05 E Exx, $xx < $E0 slide note pitch down by $xx (coarse). — E, F, G
EEx slide note pitch down by $x (extra fine).
EFx slide note pitch down by $x (fine).
$06 F Fxx, $xx < $E0 slide note pitch up by $xx (coarse). — E, F, G
FEx slide note pitch up by $x (extra fine).
FFx slide note pitch up by $x (fine).
$07 G Gxx portamento with speed $xx. — E, F, G
$08 H Hxy vibrato (pitch oscillation) with speed $x and depth $y. behaviour of only one parameter being $0 is often undefined.
standard behaviour is to use memory for each parameter that is set to $0. 		H (H, U)
$09 I Ixy tremor (toggle sample) on for $x ticks and off for $y ticks. behaviour of only one parameter being $0 is often undefined.
standard behaviour is to treat $0 as $1, unless both parameters are $0. 		I
$0A J Jxy arpeggio with the notes being root, root+$x, and root+$y. — J
$0B K K0x repeat previous vibrato + slide note volume down by $x (coarse). if neither of the digits are $0 or $F, the behaviour is undefined.
standard behaviour is to do nothing. 		K (D, K, L)
Kx0 repeat previous vibrato + slide note volume up by $x (coarse).
KFx, $0 < $x < $F repeat previous vibrato + slide note volume down by $x (fine).
KxF, $0 < $x repeat previous vibrato + slide note volume up by $x (fine).
$0C L L0x repeat previous portamento + slide note volume down by $x (coarse). if neither of the digits are $0 or $F, the behaviour is undefined.
standard behaviour is to do nothing. 		L (D, K, L)
Lx0 repeat previous portamento + slide note volume up by $x (coarse).
LFx, $0 < $x < $F repeat previous portamento + slide note volume down by $x (fine).
LxF, $0 < $x repeat previous portamento + slide note volume up by $x (fine).
$0D M Mxx set channel volume to $xx. behaviour of parameters > $40 is undefined.
standard behaviour is to do nothing. 		—
$0E N N0x slide channel volume down by $x (coarse). if neither of the digits are $0 or $F, the behaviour is undefined.
standard behaviour is to do nothing. 		N
Nx0 slide channel volume up by $x (coarse).
NFx, $0 < $x < $F slide channel volume down by $x (fine).
NxF, $0 < $x slide channel volume up by $x (fine).
$0F O Oxx start playing sample from offset $xx00. usage after SAy sets the offset to $yxx00.
behaviour of parameter $00 is often undefined if $y is nonzero.
standard behaviour is to disable memory, simply setting the offset to $y0000.
 OpenMPT: can be followed by #xx for extra digits. 		O
$10 P P0x slide channel panning right by $x (coarse). if neither of the digits are $0 or $F, the behaviour is undefined.
standard behaviour is to do nothing. 		P
Px0 slide channel panning left by $x (coarse).
PFx, $0 < $x < $F slide channel panning right by $x (fine).
PxF, $0 < $x slide channel panning left by $x (fine).
$11 Q Qxy retrigger note every $y ticks with a volume change depending on $x:
  • $0: no change
  • $1: -1
  • $2: -2
  • $3: -4
  • $4: -8
  • $5: -16
  • $6: *2/3
  • $7: *1/2
  • $8: no change
  • $9: -1
  • $A: -2
  • $B: -4
  • $C: -8
  • $D: -16
  • $E: *3/2
  • $F: *2
 behaviour of either parameter being set to $0, or $x being set to $8 is often undefined.
standard behaviour is to use memory if both parameters are $0,
treat $y as $1 if only $y is set to $0,
and simply not change the volume if $x is set to $0 or $8. 		Q
$12 R Rxy tremolo (volume oscillation) with speed $x and depth $y. behaviour of only one parameter being $0 is often undefined.
standard behaviour is to use memory for each parameter that is set to $0. 		R
$13 S
S0x — sometimes defined as the Amiga's E0x command (set resampler). S
S10 set portamento mode to smooth (default). not widely supported. only provided for backwards compatibility.
S1x, $x > $0 set portamento mode to chromatic glissando (snap to nearest notes).
S2x set sample finetune (relative to 8363 Hz). parameter is in offset binary. not widely supported. only provided for backwards compatibility. use EFx, FFx, EEx, or FEx instead.
sometimes defined as a one-digit version of XM's Wxx command (sync event).
S3x set vibrato (pitch oscillation) waveform depending on $x:
  • $0: sine (default)
  • $1: downwards saw
  • $2: square
  • $3: random/noise
 behaviour of parameters > $3 is undefined.
standard behaviour is to act like $0 (set the waveform to sine).
S4x set tremolo (volume oscillation) waveform depending on $x:
  • $0: sine (default)
  • $1: downwards saw
  • $2: square
  • $3: random/noise
S5x set panbrello (panning oscillation) waveform depending on $x:
  • $0: sine (default)
  • $1: downwards saw
  • $2: square
  • $3: random/noise
S6x extend row by $x ticks. behaviour of multiple instances (and/or SEx) on the same row is often undefined.
standard behaviour is to stack the S6x effects together, and only use the leftmost SEx effect.
S70 cut all background notes on the channel. —
S71 release all background notes on the channel.
S72 fade out all background notes on the channel.
S73 set NNA of current note to cut.
S74 set NNA of current note to continue.
S75 set NNA of current note to release.
S76 set NNA of current note to fade.
S77 disable volume envelope for the current note.
S78 enable volume envelope for the current note.
S79 disable panning envelope for the current note.
S7A enable panning envelope for the current note.
S7B disable pitch/filter envelope for the current note.
S7C enable pitch/filter envelope for the current note.
S7D force pitch/filter envelope to act as pitch envelope for the current note. OpenMPT only.
S7E force pitch/filter envelope to act as filter envelope for the current note.
S7F — —
S8x set channel panning to $x ($0 = left, $F = right). same as Xxx but with one digit of precision.
S90 disable surround panning for the channel. OpenMPT only.
S91 enable surround panning for the channel. —
S9x, $1 < $x < $8 — —
S98 disable channel reverb. OpenMPT only.
not recommended. use reverb plugins instead.
S99 enable channel reverb.
S9A set channel surround mode to centre (default). OpenMPT only.
S9B set channel surround mode to pannable.
S9C set channel filter mode to global (default).
S9D set channel filter mode to per-note.
S9E play current sample forwards.
S9F play current sample backwards.
SAx set a higher digit for subsequent sample offset commands to $x. behaviour of O00 is often undefined if $x is nonzero.
standard behaviour is to disable memory, simply setting the offset to $x0000.
SB0 mark the start of a pattern loop. —
SBx, $x > $0 jump to the start of the pattern loop until $x jumps have occured.
SCx cut the current note after $x ticks. behaviour of parameter $0 is often undefined.
standard behaviour is to treat $0 as $1.
 behaviour of parameters ≥ ticks per row is often undefined.
standard behaviour is to not cut the note at all.
SDx delay the current note by $x ticks. behaviour of parameter $0 is often undefined.
standard behaviour is to treat $0 as $1.
 behaviour of parameters ≥ ticks per row is often undefined.
standard behaviour is to not play the note at all.
SEx repeat the current row (without replaying notes) $x times. behaviour of multiple instances (and/or S6x) on the same row is often undefined.
standard behaviour is to stack the S6x effects together, and only use the leftmost SEx effect.
SFx set parametered macro bank to $x for the channel. —
$14 T T0x slide tempo down by $x (coarse). — T0 (global), T1 (global)
T1x slide tempo up by $x (coarse).
Txx, $xx ≥ $20 set tempo to $xx. OpenMPT: can be followed by #xx for extra digits.
$15 U Uxy vibrato (pitch oscillation) with speed $x and depth $y/4. behaviour of only one parameter being $0 is often undefined.
standard behaviour is to use memory for each parameter that is set to $0. 		H (H, U)
$16 V Vxx set global volume to $xx. behaviour of parameters > $80 is undefined.
standard behaviour is to do nothing. 		—
$17 W W0x slide global volume down by $x (coarse). if neither of the digits are $0 or $F, the behaviour is undefined.
standard behaviour is to do nothing. 		W (global)
Wx0 slide global volume up by $x (coarse).
WFx, $0 < $x < $F slide global volume down by $x (fine).
WxF, $0 < $x slide global volume up by $x (fine).
$18 X Xxx set channel panning to $xx ($00 = left, $FF = right). same as Xxx but with one digit of precision. —
$19 Y Yxy panbrello (panning oscillation) with speed $x and depth $y. behaviour of only one parameter being $0 is often undefined.
standard behaviour is to use memory for each parameter that is set to $0. 		Y
$1A Z Zxx, $xx < $80 run parametered macro $xx. bank can be switched for each channel using SFx. —
Zxx, $xx ≥ $80 run fixed macro $xx. —
$1B # #xx extend the parameter of the command on the previous row. OpenMPT only.
supports Bxx, Cxx, Oxx, Txx, +xx, *xx, and itself.
up to three of these commands can be chained, for a total of 8 digits. 		—
$1C \ \xx, $xx < $80 run parametered macro $xx with parameter interpolation. OpenMPT only.
bank can be switched for each channel using SFx. 		—
\xx, $xx ≥ $80 run fixed macro $xx with parameter interpolation. OpenMPT only.
$1D : :xy delay the current note by $x ticks, and cut it after $y ticks. OpenMPT only.
